# mintpop 官网 SEO 优化(SSG 预渲染)实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 用 `vite-ssg` 构建期预渲染 mintpop 落地页,并补齐每页 Meta、OG/Twitter 卡片、JSON-LD、sitemap、robots,在不改部署模式的前提下完成 SEO 优化。

**Architecture:** 把入口从 `createApp(App).mount()` 换成 `ViteSSG(App, { routes }, setup)`,构建期对每个路由渲染出真实 HTML,客户端自动 hydration。head 由 `@unhead/vue`(vite-ssg 原生集成)管理。部署产物仍是 `dist/` 静态文件,nginx/Docker/compose/CI 全部不动。

**Tech Stack:** Vue 3 + Vite 6 + TypeScript + vite-ssg + @unhead/vue + pnpm(经 mise)。

## Global Constraints

- 工具链与命令统一走 mise:依赖装/构建用 `mise run install` / `mise run build`,底层 pnpm。
- 所有新增代码注释、文档、提交信息用**简体中文**;代码标识符保持英文。
- 生产域名常量唯一来源:`SITE_URL = 'https://mintpop.ai'`,放 `src/config/site.ts`,全站从此取。

```ts
/** 读取初始主题:SSR 阶段无浏览器全局,回退浅色;客户端按 localStorage→系统偏好→浅色 */
function resolveInitialTheme(): Theme {
  if (typeof window === 'undefined') {
    return Theme.LIGHT
  }
  const saved = localStorage.getItem(STORAGE_KEY_THEME)
  if (saved === Theme.LIGHT || saved === Theme.DARK) {
    return saved
  }
  const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches
  return prefersDark ? Theme.DARK : Theme.LIGHT
}
```

```ts
/** 把当前主题与主题色同步到 :root 上(CSS 变量驱动整页配色);SSR 阶段无 document,直接跳过 */
function applyToDocument(): void {
  if (typeof document === 'undefined') {
    return
  }
  const root = document.documentElement
  root.dataset.theme = theme.value
```

- [ ] **Step 3: 创建 `src/composables/useSeo.ts`(基础版,仅 Meta/canonical)**

```ts
import { useHead } from '@unhead/vue'

import { absoluteUrl } from '@/config/site'
import { SEO, type SeoRouteKey } from '@/config/seo'

/**
 * 按路由 key 声明该页 head:title、description、canonical;
 * noindex 页额外输出 robots=noindex。Task 3 将在此扩展 OG/Twitter/JSON-LD。
 */
export function useSeo(key: SeoRouteKey): void {
  const entry = SEO[key]
  const canonical = absoluteUrl(entry.path)

  const meta = [
    { name: 'description', content: entry.description },
  ]
  if (entry.noindex) {
    meta.push({ name: 'robots', content: 'noindex' })
  }

  useHead({
    title: entry.title,
    meta,
    link: [{ rel: 'canonical', href: canonical }],
  })
}
```

- [ ] **Step 3: 在 `useSeo` 补 OG/Twitter 标签**

整文件替换 `src/composables/useSeo.ts`:

```ts
import { useHead } from '@unhead/vue'

import { SEO, type SeoRouteKey } from '@/config/seo'
import { absoluteUrl, OG_IMAGE_PATH, SITE_NAME } from '@/config/site'

/**
 * 按路由 key 声明该页 head:基础 Meta + canonical + OG + Twitter 卡片。
 * noindex 页额外输出 robots=noindex。
 */
export function useSeo(key: SeoRouteKey): void {
  const entry = SEO[key]
  const canonical = absoluteUrl(entry.path)
  const ogImage = absoluteUrl(OG_IMAGE_PATH)

  const meta = [
    { name: 'description', content: entry.description },
    // Open Graph
    { property: 'og:type', content: 'website' },
    { property: 'og:site_name', content: SITE_NAME },
    { property: 'og:locale', content: 'en_US' },
    { property: 'og:title', content: entry.title },
    { property: 'og:description', content: entry.description },
    { property: 'og:url', content: canonical },
    { property: 'og:image', content: ogImage },
    // Twitter
    { name: 'twitter:card', content: 'summary_large_image' },
    { name: 'twitter:title', content: entry.title },
    { name: 'twitter:description', content: entry.description },
    { name: 'twitter:image', content: ogImage },
  ]
  if (entry.noindex) {
    meta.push({ name: 'robots', content: 'noindex' })
  }

  useHead({
    title: entry.title,
    meta,
    link: [{ rel: 'canonical', href: canonical }],
  })
}
```

### Task 4: sitemap.xml + robots.txt

构建期生成 sitemap(仅可收录路由),并加 robots.txt。

**Files:**
- Create: `public/robots.txt`
- Modify: `vite.config.ts`(`ssgOptions.onFinished` 写 sitemap)

**Interfaces:**
- Consumes: Task 2 的 `src/config/site.ts` 的 `SITE_URL` 与 `INDEXABLE_ROUTES`。
- Produces: 构建产物 `dist/sitemap.xml`、`dist/robots.txt`。

- [ ] **Step 1: 创建 `public/robots.txt`**

```text
User-agent: *
Allow: /
Disallow: /login
Disallow: /register

Sitemap: https://mintpop.ai/sitemap.xml
```

- [ ] **Step 2: 在 `vite.config.ts` 顶部静态 import 常量与 writeFile**

在文件顶部的 import 区(`/// <reference ... />` 之后)加入两行 —— vite 加载 config 时会用 esbuild 把这两个 import 一起打包,避免运行时 Node 直接导入 `.ts` 的问题,同时复用 `site.ts` 常量保持单一来源:

```ts
import { writeFile } from 'node:fs/promises'

import { INDEXABLE_ROUTES, SITE_URL } from './src/config/site'
```

- [ ] **Step 3: 在 ssgOptions 加 onFinished 生成 sitemap**

把 `ssgOptions` 替换为:

```ts
  // vite-ssg 预渲染选项
  ssgOptions: {
    script: 'async',
    formatting: 'minify',
    async onFinished() {
      const body = INDEXABLE_ROUTES.map(
        (path) => `  <url><loc>${SITE_URL}${path}</loc></url>`,
      ).join('\n')
      const xml = `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
${body}
</urlset>
`
      await writeFile('dist/sitemap.xml', xml)
      console.log('Sitemap written to dist/sitemap.xml')
    },
  },
```

- [ ] **Step 4: 构建并验证 sitemap / robots**

Run:
```bash
mise run build && \
test -f dist/sitemap.xml && echo "SITEMAP_EXISTS" && \
[ "$(grep -c '<loc>' dist/sitemap.xml)" = "2" ] && echo "SITEMAP_COUNT_OK" && \
grep -q '<loc>https://mintpop.ai/</loc>' dist/sitemap.xml && echo "SITEMAP_HOME_OK" && \
! grep -q '/login' dist/sitemap.xml && echo "SITEMAP_NO_LOGIN_OK" && \
test -f dist/robots.txt && grep -q 'Sitemap: https://mintpop.ai/sitemap.xml' dist/robots.txt && echo "ROBOTS_OK"
```
Expected: 打印 `SITEMAP_EXISTS`、`SITEMAP_COUNT_OK`、`SITEMAP_HOME_OK`、`SITEMAP_NO_LOGIN_OK`、`ROBOTS_OK`。
